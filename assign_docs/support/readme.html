Running the Relay Server:
==============
Use the following command

    java -jar rserver.jar

to start the server. To stop the server, type "quit" on the screen. To change the
network topology, type "update" on the screen. In response to command "update",
the relay server cycles through the list of topologies defined in the topology file. If
it reaches the last topology then it moves back to the first topology.

Using the above command, the server starts with a set of default parameters.
Specifically, it assumes that the topology file name is "topology.txt".
You can run the server in the following modes to specify your own parameters.

1) Non-verbose Mode:

    java -jar rserver.jar port topology loss

where,

port: the server local port number.

topology: the name of the topology file.

loss: the packet loss probability at the server.

It specifies the ratio of lost packets at the server. The server randomly
drops arriving packets. The probability that the server drops a packet
is given by this parameter.
 

2) Verbose Mode:

    java -jar rserver.jar -v port topology loss
 

By specifying the option '-v', the server works in verbose mode, where
it prints detailed information regarding received routing packets.
